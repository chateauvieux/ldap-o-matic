Configuration
SUPPORTED PLATFORMS

MIG runs on any platforms supporting the Apache Web Server (that is to say, on most UNIX and Windows platforms). MIG is currently tested against OpenLDAP directories.

REQUIREMENTS

MIG requires : 

the Apache Web Server with PHP support
A LDAP-version 2 or 3 compliant server, preferably supporting SSL and Strong Authentication. 
Apache/PHP requirements details
Apache version must be superior to 1.3.12

MIG is currently tested against Apache 1.3.12 and 1.3.19. It may work with prior versions but this has not been tested.

PHP version 4.0.5 or above is required. The module must be installed with the following features :
automatic session ID propagation (--enable-trans-sid configure option)
GD image libraries (--with-gd)
expat library for UTF8 support (--with-xml)
SUDO for allowing the user under which the web daemon is run to restart the LDAP service.

MIG is tested with a PHP module configured as follow: ./configure --enable-trans-sid --with-mysql --with-ldap --with-openssl=/usr/local/ssl --with-apxs --with-gd. Cookies can be disabled. Error reporting can be turned on to maximum value.

Supported LDAP servers

Currently supported servers are :

OpenLDAP 2.0.x
OpenLDAP 1.2.x
 
INSTALLATION
Install Apache web server - www.apache.org
Install the PHP module - www.php.net.
Minimum supported version is php 4.0.5
Warning: PHP 4.1 (up to 4.1.2) introcuded some backwards-compatibility issues and is not yet supported
Make sure .html pages will be processed by PHP. In the apache configuration file (httpd.conf) check for the directive
Make sure magic_quotes_gpc is turned of in the php.ini file
AddType application/x-httpd-php .php4 .html .htm
Move MIG files in a web directory, and ensure that the web server's PHP module is running. 
Import the MIG basic directory skeleton file (MIG_skel.ldif) in your directory. 
Create a web folder for MIG. Uncompress the MIG-common.tar.gz archive into this folder.
Uncompress the MIG-site-specific.tar.gz into a separate directory. Edit the MIG_install.sh orig and mig_dir variable to fit your installation paths
Example:

orig=/home/krist/MIG_uwc_specific
mig_dir=/home/uwcnet/html/MIG

Edit the configuration file (config.php) according to your system settings.
Run the MIG_install.sh post-installation script


MIG depends on
a logfile (see MIG_site_specific/config.php)
a read-protected temporary folder containing session information 
(see MIG_site_specific/config.php and the session.use_cookies and session.save_path directives of the phpini file from PHP)
persmissions on this folder should be rw- --- ---, owner being the UID under which the web server operates
the LDAP server's schema file where MIG will append user-defined attributes and objectclasses definitions (see MIG_site_specific/config.php)
Metamerge Integrator if configured to used MI (see MIG_site_specific/config.php)
create a user for Integrator
install JAVA, install Integrator
run the init.d script to startup on Linux systems
sudo and its /etc/sudoers files configured to allow a LDAP server restart from the web UID.

That's it. MIG should be running !
