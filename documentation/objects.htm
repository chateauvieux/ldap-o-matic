Your information

MIG distinguishes every piece of information by the "something" it is the image of; for instance, it could be  a day, an age, a temperature, etc. which value could be 20 for instance. Each information atom in MIG is therefore accessed through an attribute name, and leads to the information itself, stored into the low-level information system. An attribute in MIG can therefore be seen as a link, and with directory technology, it is basically a pointer to an LDAP attribute with some additional information. 

Moreover, MIG allow you to manage search classes, that is to say a subset of attributes. For example, we can imagine that an "employees" search class will consist of employees names, date of arrival, salary information, email address, etc. Search classes are such as "attribute groups", and are used to allow information searching or editing. A search class in MIG can therefore be seen as a group of attributes, and with directory technology, it is basically a pointer to an LDAP objectclass with some additional information. 

In order to be accessed, those search classes may need authentication. For that reason, each search class can be set as public or private. In the latter case, only users that get authorization can access the data encapsulated in the search class.

 

Users

MIG can be used by two categories of actors, who must authenticate when entering the web interface :

ordinary users, which will have the opportunity of editing their own data and browsing search classes (if they have the permission to do so),
administrators, which have super-user grants for browsing but also access to administration tools, such as user accounts management, attribute and search classes management, etc.
