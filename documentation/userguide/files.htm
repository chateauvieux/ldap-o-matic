The MIG system consists of several PHP include files containing objects definitions, and HTML pages that load those objects and make call to their methods. Here are the include files and their utility for an LDAP configuration:

File name Description Requires
php/config.php MIG configuration file. Every object relies on this file so it must be loaded before any other.   
php/ldap_core.php MIG's LDAP core layer file. php/config.php
php/attribute.php MIG's attribute definition php/config.php
php/ldap_core.php
php/search_class.php
php/search_class.php

 

 MIG's search class definition php/config.php
php/ldap_core.php
php/attribute.php
php/user.php

 

 MIG's user definition php/config.php
php/ldap_core.php
php/group.php MIG's group definition php/config.php
php/ldap_core.php

When you want to use one of the objects, load its include file thanks to the require keyword. Do not forget to include the files required by the include files. 

For example, if you want to use the attribute object in an LDAP context, you first have to insert this code:

require "php/config.php";
require "php/ldap_core.php";
require "php/search_class.php";
require "attribute.php";
The context is set in MIG's configuration file, so a more generic way to achieve this is to insert the following:
require "php/config.php";
$core_file = "php/" . INFORMATION_SYSTEM_TYPE . "_core.php";
require $core_file;
require "php/search_class.php";
require "attribute.php";
