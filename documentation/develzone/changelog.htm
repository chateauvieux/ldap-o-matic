MIG Changelog

version 1.6

Fixed security holes:
MIG - to - integrator communication
ACLs
MIG now uses the OpenLDAP ACL's (including sets, dynamic filtering etc) and is able to reconfigure OpenLDAP.
Fixed a bug with dn attributes when space characters are present before/after the RDN separator
Improved GUI (at last!)
Fixed a bug in the logon mechanism

version 1.5

Added support for password encryption. MIG can now encrypt password in SSHA, MD5 and CRYPT. The two first options only if PHP is compiled with the mhash option.

version 1.4

Added MIG_INSTALLATION_NAME as a parameter.
Added MIG_FORCE_FRAMES as a parameter. Useful if you want to have MIG running as a portal.
New error handling scheme. Added MIG_LOGFILE.
Improved the attributes management pages. Lighter processing for NAMES/VALUES and TYPES views.
Added a "Add record" page for both Metamerge Integrator and Integrator-less modes.
Fixed a bug that displayed incorrectly groups editable by delegated administrators.

 

version 1.3

MIG can now run without Metamerge Integrator. It can be pure PHP but then misses the synchronisation and provisioning capabilities offered by the Java tool. Moreover it will only be able to modify one server at a time. Added MIG_USES_INTEGRATOR as a parameter.
Search classes now have additional properties
Bug fixing for special characters (norwegian, chinese..): they are now encoded correctly. MIG assumes UTF8 (can be set in the MIG config file).
LDAP v3 and empty strings: PHP now removes empty strings when talking to a directory. In Integrator-less mode that means that attributes set to "" will NOT be removed. The Integrator behaviour is unchanged.
Fixed a nasty bug for the LDAP schema module. It now works fine.
Added automatic OID discovery for standard attributes.

version 1.2

GUI works - New feature: subsections allows now to control the order in which the attributes of a search class would be displayed in the entry view/edit window. Limitated in functionality, especially for inserts... I may get help on GUI soon...

Version 1.1

MIG is now able to do an LDAP schema reconfiguration (RFC 2252 compliant) on OpenLDAP > 2.0.14 servers. It writes any custom (i.e user-defined) attribute in a schema file for the server to load. Added MIG_LDAP_SERVER_INSTALLATION_SCHEMA_INCLUDE_FILE parameter for a local or NFS-mounted file (ftp is not yet supported).
MIG Setup page (html) started...

version 1.00

Tested for referency checks.
changed configuration file. Added USERS_MIG_SEARCHCLASS, USERS_DOMAIN and MIG_SESSION_TIMEOUT parameters.
added SSL support. MIG requires now SSL to be activated in the PHP module.
added attribute edit, display type and search type properties : current supported types are available in the attribute object documentation, (date ,drop-down menu, four digit text box...). A special type called "combination" enables attributes combinations. In this case, attribute migauthorizedvalues refer to attribute names and string formatting is supported as well thanks to the #STRING keyword.
Image generation support. MIG requires now the GD librairies to be compiled with PHP.
Attributes ID have not to be lowercase anymore.
Fixed a bug in the attribute object that caused properties flush when changing attribute ID.
Fixed a bug in the group object that caused properties flush when changing attribute ID.
added attribute migdescription and example properties, to allow creation of attribute help pages.

authentication process optimization (60% faster)
added group object which gives to MIG users the ability of managing information assets. Added , and group administration page.
unpatched ldap_core as PHP 4.0.4 is released
added file inclusion support for attribute migauthorizedvalues (#INCLUDE keyword)
one small bug fix in LDAP core file
